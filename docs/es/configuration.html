Configuracion

En la primera ejecucion, vtmate crea ~/.vtmate/settings con dos agentes si el archivo no existe. Puedes definir tantos agentes como quieras, cambiarlos en tiempo real con las flechas, cargar archivos alternativos con -c y listar voces con vtmate --list-voices.

Ejemplo de configuracion de agente

[agent]
name = main agent
language = en
voice = bf_alice
voice_speed = 1.1
provider = ollama
baseurl = http://localhost:8080
model = gpt-oss-20b
system_prompt = You are a smart ai assistant.
sound_threshold_peak = 0.1
end_silence_ms = 2000
tts = kokoro
ptt = false
whisper_model_path = ~/.whisper-models/ggml-tiny.bin

Cargar un archivo de configuracion alternativo

vtmate -c philosophers-settings.txt

Listar voces disponibles

vtmate --list-voices

Detalles de los campos de configuración

  • name: un nombre corto para el agente
  • language: cualquiera de los idiomas disponibles usados para reconocimiento de voz y TTS
  • voice: el nombre de la voz que usará el agente (ver voces disponibles para cada idioma y sistema TTS ejecutando `ai-mate --list-voices`). Mezcla de voces: puedes mezclar 2 voces. ejemplo de mezcla 50% de bm_daniel y 50% de am_puck: "bm_daniel.5+am_puck.5"
  • voice_speed: la velocidad de la voz de 1.0 a 9.0
  • provider: el sistema que usará para consultar el LLM, puede ser 'ollama' o 'llama-server'
  • baseurl: la url base usada para contactar al proveedor (debe ser sin ruta)
  • model: el nombre del modelo a usar en ollama (algunas versiones de llama-server ignorarán esta opción ya que llama-server corre con un solo modelo)
  • system_prompt: el prompt del sistema que se enviará al LLM al consultarlo. Usa \n para nuevas líneas
  • sound_threshold_peak: un valor entre 0 y 1 que se usará como base de pico para detectar el habla del usuario
  • end_silence_ms: los milisegundos de silencio por debajo del nivel de sound_threshold_peak que deben transcurrir para que el habla del usuario sea enviada. En modo ptt, esta opción se ignora, el programa esperará que se suelte la tecla ESPACIO para enviar el audio.
  • tts: el sistema TTS a usar, puede ser 'kokoro' o 'opentts'. opentts requiere que el contenedor docker de opentts esté corriendo: docker run -p 5500:5500 synesthesiam/opentts:all
  • ptt: modo push-to-talk, cuando está establecido en true debes mantener presionada la barra espaciadora mientras hablas, luego suelta.
  • whisper_model_path: la ruta al modelo whisper. ai-mate descomprime 2 modelos en ~/.whisper-models, tiny y small. Puedes descargar modelos más grandes y apuntar a ellos aquí. Asegúrate de que esté correctamente traducido para cada idioma
