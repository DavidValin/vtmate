Documentacion

El sistema final de conversacion por voz con IA ejecutandose en tu terminal. Esta pagina cubre instalacion, configuracion y los modos principales de uso.

ConfiguracionConfigura agentes, voces, proveedores y archivos de ajustes. Modo ConversacionEjecuta el bucle de voz en vivo y controlalo con el teclado. Modo DebateHaz que dos agentes hablen entre si desde una entrada inicial. Modo LecturaLee archivos de texto o stdin con la voz del agente seleccionado. Modo Ejecucion UnicaObtiene una respuesta y termina para automatizacion o shell.

Configuracion

En la primera ejecucion, vtmate crea ~/.vtmate/settings con dos agentes si el archivo no existe. Puedes definir tantos agentes como quieras, cambiarlos en tiempo real con las flechas, cargar archivos alternativos con -c y listar voces con vtmate --list-voices.

Ejemplo de configuracion de agente

[agent]
name = main agent
language = en
voice = bf_alice
voice_speed = 1.1
provider = ollama
baseurl = http://localhost:8080
model = gpt-oss-20b
system_prompt = You are a smart ai assistant.
sound_threshold_peak = 0.1
end_silence_ms = 2000
tts = kokoro
ptt = false
whisper_model_path = ~/.whisper-models/ggml-tiny.bin

Cargar un archivo de configuracion alternativo

vtmate -c philosophers-settings.txt

Listar voces disponibles

vtmate --list-voices

Modo Conversacion

El bucle en vivo es RECORD -> STT -> LLM -> TTS -> PLAYBACK. La grabacion empieza cuando el audio supera el umbral configurado y termina con silencio, salvo si usas push-to-talk. Puedes pausar la grabacion con SPACE, parar la reproduccion con un ESCAPE, interrumpir la respuesta completa con dos ESCAPE, cambiar de agente con ARROW_LEFT/ARROW_RIGHT y ajustar la velocidad de voz con ARROW_UP/ARROW_DOWN.

Conversacion por defecto y guardar audio mas texto

vtmate -s

Iniciar con un agente especifico

vtmate --agent "main agent"

Iniciar con un prompt desde la CLI

vtmate -p "¿Estamos solos en la galaxia?"

Iniciar con un prompt desde archivo

vtmate -i myprompt.txt

Iniciar con un prompt desde stdin

echo "¿Cómo volar sin alas?" | vtmate -i -

Modo Debate

El modo debate permite que dos agentes hablen entre si a partir de una entrada inicial. Es util para razonamiento adversarial o exploracion por roles. Puedes interrumpir y cambiar el tema en cualquier momento. Desde el modo conversacion tambien puedes iniciar o parar un debate con Control+D.

Iniciar un debate desde un tema inicial

vtmate --debate "God" "Devil" "¿Cómo tener éxito en la vida?"

Iniciar un debate con prompt desde archivo

vtmate --debate "God" "Devil" -i myprompt.txt

Iniciar un debate con prompt desde stdin

echo "Lets discuss the permissions of this files:" | vtmate --debate "Unix administrator" "Security Expert" -i -

Guardar el debate como audio mas texto

vtmate --debate "Aristoteles" "Ptahhotep" "¿Quién fue el primer filósofo?" -s

Modo Lectura

El modo lectura convierte texto desde archivo o stdin en salida hablada frase por frase usando la voz del agente seleccionado. En este modo, los ajustes relevantes del agente son tts, voice y language. Usa ARROW_UP y ARROW_DOWN para moverte entre frases, y SPACE para parar o reanudar la reproduccion.

Leer desde un archivo de texto

vtmate -r myfile.txt --agent "main agent"

Leer desde stdin

echo "Hola. ¿Cómo estas?" | vtmate -r - --agent "main agent"

Modo Ejecucion Unica

El modo de ejecucion unica obtiene una respuesta y termina. Es util para automatizacion, pipelines de shell y scripts. Puedes proporcionar el prompt directamente, desde archivo o desde stdin, y guardar el resultado como audio y texto.

Ejecutar un prompt directo

vtmate -q -p "Explicame el metodo Zettelkasten"

Ejecutar un prompt desde archivo

vtmate -q -i myprompt.txt

Ejecutar un prompt desde stdin

echo "¿Es festivo nacional el $(date) en españa?" | vtmate -q -i -

Guardar el resultado como audio mas texto

echo "¿Puedes encontrar algún proceso sospechoso? Si es asi, por qué?\n\n $(ps aux | head -20)" | vtmate -q -i - -s
